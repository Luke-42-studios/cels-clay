<objective>
Wrap the existing NCurses renderer in a CEL_Module and establish renderer infrastructure with CMake conditional compilation.

Purpose: The NCurses renderer currently uses a manual init function (clay_ncurses_renderer_init). Wrapping it in CEL_Module(Clay_NCurses) enables renderer selection via module registration (`cels_register(Clay_NCurses)`) and aligns with the cels-sdl3 module pattern. The CMake infrastructure adds the placeholder for the Phase 10 SDL3 renderer.

Output: Modernized clay_ncurses_renderer.h/c with CEL_Module(Clay_NCurses), configure function for theme, CMake guards for both renderer backends.
</objective>

Migrate the NCurses renderer from manual init to CEL_Module pattern.

**clay_ncurses_renderer.h changes:**

1. Add CEL_Module declaration:
   ```c
   CEL_Module(Clay_NCurses);
   ```

2. Replace `clay_ncurses_renderer_init()` with a pre-init configure function:
   ```c
   /* Configure NCurses Clay renderer theme before module registration.
    * Call before cels_register(Clay_NCurses). Pass NULL for default theme.
    * If not called, default theme is used. */
   extern void Clay_NCurses_configure(const ClayNcursesTheme* theme);
   ```

3. Keep `clay_ncurses_renderer_set_theme()` for runtime theme changes.

4. Keep `clay_ncurses_handle_scroll_input()` -- it's a utility function, not tied to module pattern.

5. Keep `ClayNcursesTheme` struct and `CLAY_NCURSES_THEME_DEFAULT` unchanged.

6. Remove `clay_ncurses_renderer_init()` declaration.

7. Update header comment to describe module registration pattern:
   ```
   Usage:
     #include <cels-clay/clay_ncurses_renderer.h>

     CEL_Build(App) {
         cels_register(Clay_Engine, Clay_NCurses);
         // Optional: Clay_NCurses_configure(&custom_theme); before register
     }
   ```

**clay_ncurses_renderer.c changes:**

1. Add `CEL_Module(Clay_NCurses, init)` body that does what `clay_ncurses_renderer_init()` currently does:
   ```c
   CEL_Module(Clay_NCurses, init) {
       // Use stored theme (from Clay_NCurses_configure) or default
       if (!g_theme) g_theme = &CLAY_NCURSES_THEME_DEFAULT;

       // Register text measurement callback
       Clay_SetMeasureTextFunction(clay_ncurses_measure_text, NULL);

       // Register render system at OnRender phase
       ClayRenderableData_register();
       cels_entity_t comp_ids[] = { ClayRenderableData_id };
       cels_system_declare("TUI_ClayRenderable_ClayRenderableData",
                           CELS_Phase_OnRender, clay_ncurses_render, comp_ids, 1);
   }
   ```

2. Add `Clay_NCurses_configure()`:
   ```c
   void Clay_NCurses_configure(const ClayNcursesTheme* theme) {
       g_theme = theme ? theme : &CLAY_NCURSES_THEME_DEFAULT;
   }
   ```

3. Remove `clay_ncurses_renderer_init()` function body (replaced by module init).

4. All render helper functions (render_rectangle, render_text, render_border, render_border_decor, clay_ncurses_render, clay_ncurses_measure_text, coordinate mapping, scroll input) remain unchanged.

5. The `g_theme` initialization changes from `= &CLAY_NCURSES_THEME_DEFAULT` to `= NULL` so that Clay_NCurses_configure can be detected. The module body checks for NULL and falls back to default.

**Developer usage pattern changes:**

Before (v0.5):
```c
Clay_Engine_use(NULL);
clay_ncurses_renderer_init(NULL);  // or with theme
```

After (v0.6):
```c
// Optional: Clay_NCurses_configure(&my_theme);
cels_register(Clay_Engine, Clay_NCurses);
```
