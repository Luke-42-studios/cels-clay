<objective>
Create entity primitive components and compositions for Row, Column, Box, Text, Spacer, and Image.

Purpose: These are the building blocks developers use to construct UI. Instead of writing layout functions with raw CLAY() macros, developers compose entities like `ClayRow(.gap = 2) { ClayText(.text = "Hello") {} }`. The components carry per-entity properties that Phase 8's layout walker reads to generate CLAY() calls.

Output: clay_primitives.h with 4 config component structs (Container, Text, Spacer, Image), 1 additive component (BorderStyle), 6 static-linkage compositions with call macros, and clay_primitives.c with component registration.
</objective>

**Component structs (4 config components + 1 additive):**

1. **ClayContainerConfig** -- shared by Row, Column, and Box:
   ```c
   typedef struct ClayContainerConfig {
       Clay_LayoutDirection direction;  // LEFT_TO_RIGHT (Row) or TOP_TO_BOTTOM (Column/Box)
       uint16_t gap;
       Clay_Padding padding;
       Clay_SizingAxis width;
       Clay_SizingAxis height;
       Clay_ChildAlignment alignment;  // contains .x and .y
       Clay_Color bg;
       bool clip;
   } ClayContainerConfig;
   ```
   Use `Clay_ChildAlignment` (which has .x and .y fields) instead of separate main/cross alignment fields. This is simpler and maps directly to Clay's `.layout.childAlignment`.

2. **ClayTextConfig**:
   ```c
   typedef struct ClayTextConfig {
       const char* text;
       Clay_Color color;
       uint16_t font_size;
       uint16_t font_id;
       uint16_t letter_spacing;
       uint16_t line_height;
       Clay_TextElementConfigWrapMode wrap;
   } ClayTextConfig;
   ```

3. **ClaySpacerConfig**:
   ```c
   typedef struct ClaySpacerConfig {
       Clay_SizingAxis width;
       Clay_SizingAxis height;
   } ClaySpacerConfig;
   ```

4. **ClayImageConfig**:

       float source_height;
       Clay_SizingAxis width;
       Clay_SizingAxis height;

**Static-linkage compositions** (same pattern as ClaySurface in clay_layout.h):

For INTERFACE library compatibility, compositions use the manual static expansion pattern (not CEL_Define_Composition which generates extern symbols that would conflict across TUs).

Each composition:
1. Defines a Props typedef with the relevant fields
2. Has a static `_Impl` function that calls `cel_has(ComponentType, ...)` to attach the component
3. Has a static `_factory` wrapper
4. Has a `#define Name(...) cel_init(Name, __VA_ARGS__)` call macro

**ClayRow composition:**
```c
typedef struct ClayRowProps {
    cels_lifecycle_def_t* lifecycle;
    uint16_t gap;
    Clay_Padding padding;
    Clay_SizingAxis width;
    Clay_SizingAxis height;
    Clay_ChildAlignment alignment;
    Clay_Color bg;
    bool clip;
} ClayRowProps;

static void ClayRow_Impl(ClayRowProps props) {
    cel_has(ClayContainerConfig,
        .direction = CLAY_LEFT_TO_RIGHT,
        .gap = props.gap,
        .padding = props.padding,
        .width = props.width,
        .height = props.height,
        .alignment = props.alignment,
        .bg = props.bg,
        .clip = props.clip
    );
}
// ... factory wrapper ...
#define ClayRow(...) cel_init(ClayRow, __VA_ARGS__)
```

**ClayColumn composition:** Same as ClayRow but sets `.direction = CLAY_TOP_TO_BOTTOM`.

**ClayBox composition:** Same as ClayColumn (TOP_TO_BOTTOM), but with additional border/corner_radius fields in its props that attach a ClayBorderStyle component:
```c
typedef struct ClayBoxProps {
    cels_lifecycle_def_t* lifecycle;
    Clay_Padding padding;
    Clay_SizingAxis width;
    Clay_SizingAxis height;
    Clay_ChildAlignment alignment;
    Clay_Color bg;
    Clay_Border border;
    Clay_CornerRadius corner_radius;
    bool clip;
} ClayBoxProps;

static void ClayBox_Impl(ClayBoxProps props) {
    cel_has(ClayContainerConfig,
        .direction = CLAY_TOP_TO_BOTTOM,
        .padding = props.padding,
        .width = props.width,
        .height = props.height,
        .alignment = props.alignment,
        .bg = props.bg,
        .clip = props.clip
    );
    // Attach border style only if border is non-zero
    // (check any width field)
    if (props.border.width.top || props.border.width.right ||
        props.border.width.bottom || props.border.width.left) {
        cel_has(ClayBorderStyle,
            .border = props.border,
            .corner_radius = props.corner_radius
        );
    }
}
```

**ClayText composition:**
```c
typedef struct ClayTextProps {
    cels_lifecycle_def_t* lifecycle;
    const char* text;
    Clay_Color color;
    uint16_t font_size;
    uint16_t font_id;
    uint16_t letter_spacing;
    uint16_t line_height;
    Clay_TextElementConfigWrapMode wrap;
} ClayTextProps;

static void ClayText_Impl(ClayTextProps props) {
    cel_has(ClayTextConfig,
        .text = props.text,
        .color = props.color.r || props.color.g || props.color.b || props.color.a
            ? props.color : (Clay_Color){255,255,255,255},
        .font_size = props.font_size ? props.font_size : 16,
        .font_id = props.font_id,
        .letter_spacing = props.letter_spacing,
        .line_height = props.line_height,
        .wrap = props.wrap
    );
}
```
